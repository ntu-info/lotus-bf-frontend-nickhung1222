# LoTUS-BF Frontend

這是一個為 **LoTUS-BF (Location-or-Term Unified Search for Brain Functions)** 設計的前端應用程式介面。它旨在提供一個高效且直觀的平台，讓神經科學研究人員可以透過關鍵術語 (Term) 或 MNI 座標 (MNI coordinates) 來搜尋相關的腦功能研究文獻。

---

## ✨ 核心功能

- **三欄式互動介面**：將術語瀏覽、查詢建立與結果，以及腦圖可視化三大核心功能整合在同一個畫面上，並支援使用者自由拖動調整各區塊寬度。
- **術語瀏覽與點選**：使用者可以從左側面板瀏覽或搜尋術語，並點選加入查詢。
- **動態查詢建立**：在中間的查詢建立器 (Query Builder) 中，使用者可以混合使用術語和邏輯運算子 (AND, OR, NOT) 來建立複雜的查詢。
- **文獻結果呈現**：根據查詢結果，在中間面板下方即時呈現相關的學術研究列表。
- **腦圖可視化**：在右側面板，提供 Nii 檔案的檢視器，用於視覺化腦部影像資料。

## 🚀 近期 UX/UI 優化亮點

在原有基礎上，我們合作新增了以下幾個提升使用者體驗 (User Experience) 的功能：

1.  **可收合的側邊面板**
    - **功能**：在「Terms」面板的標題欄新增了一個圖示按鈕，允許使用者收合或展開此面板。
    - **UX 價值**：當使用者需要專注於查詢結果時，可以收合側邊面板以獲得更寬敞的閱讀空間。我們還將按鈕的行為優化為「固定位置」，避免了操作後按鈕位置改變帶來的困惑。

2.  **一鍵重設版面**
    - **功能**：在頁面頂部新增了一個「Reset Layout」按鈕。
    - **UX 價值**：讓使用者可以安心地自由拖動、客製化版面，因為他們知道隨時可以透過這個按鈕一鍵還原到預設的最佳佈局，提升了操作的安全感和效率。

3.  **已選術語高亮**
    - **功能**：當使用者從「Terms」列表中點選一個術語加入查詢後，該術語會在列表中以深藍色背景高亮顯示。
    - **UX 價值**：提供了清晰的視覺回饋，讓使用者可以快速對照，了解哪些術語已經在當前的查詢中，避免重複添加或遺漏。

4.  **介面圖示化**
    - **功能**：將「收合/展開」等功能按鈕從純文字改為更直觀的箭頭圖示。
    - **UX 價值**：圖示能讓使用者更快地識別功能，降低介面的認知負擔，並提升了整體的視覺質感與現代感。

## 🛠️ 如何在本機運行

1.  **安裝 Node.js**:
    - 建議使用 `nvm` (Node Version Manager) 來安裝並管理 Node.js 版本。
    - `nvm install --lts`

2.  **安裝專案依賴**:
    - 如果遇到問題，可以先刪除舊的依賴：`rm -rf node_modules package-lock.json`
    - `npm install`

3.  **啟動開發伺服器**:
    - `npm run dev`
    - 伺服器將會運行在 `http://localhost:5173`。

4.  **打包用於部署**:
    - `npm run build`
    - 打包後的檔案會生成在 `dist` 資料夾中。
